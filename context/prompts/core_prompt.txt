## Task: Translate a component to i18n

Target file: [COMPONENT PATH]

Steps:
1. Read the file and list every hardcoded English string.
2. Design key names using the convention:
   btn_<action> | lbl_<field> | msg_<context> | placeholder_<field> | <module>_title
3. Add `const { t } = getTranslation()` at the TOP of each component function body.
4. Replace all hardcoded strings with `t('key')`.
5. Add all new keys to public/locales/en.json AND public/locales/bn.json.
6. Validate both JSON files with:
   python3 -c "import json; json.load(open('public/locales/en.json'))"
   python3 -c "import json; json.load(open('public/locales/bn.json'))"
7. Grep the component to confirm no t('key') calls reference missing keys.
8. Mark the component done in context/TASKS.md.

CRITICAL RULES:
- getTranslation() must be called at component top level — NEVER inside .map(), useCallback, useMemo, or helpers.
- Add keys to BOTH locale files or the raw key will appear on screen in one language.
- Use showConfirmDialog / showSuccessDialog / showErrorDialog from @/lib/toast — never SweetAlert2 directly.

## Task: Debug a raw key showing on screen

Symptom: A raw key string like `purchase_title` appears in the UI instead of translated text.

Steps:
1. grep -rn "purchase_title" app/ components/ --include="*.tsx" — find the component
2. python3 -c "import json; d=json.load(open('public/locales/en.json')); print(d.get('purchase_title','MISSING'))"
3. If MISSING → add to both locale files
4. If present → check getTranslation() placement in the component (must be at top of function body)
